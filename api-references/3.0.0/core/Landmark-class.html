Landmark class.

Represents a predefined, permanent place enriched with detailed metadata, useful for search, display, and route calculation. A landmark may refer to a building, point of interest (POI), region, or settlement or a custom userâ€‘defined place.

Landmarks can originate from:

  • A LandmarkStore
  • Search results via SearchService
  • Custom creation or GeoJSON import
  • Selection from GemMapController

Features:

  • Geographic context: coordinate and multi-type boundary
  • Descriptive fields: localized name, description, author/attribution, timestamp
  • Metadata: categories (LandmarkCategory), key/value extras, contact details, store assignment info
  • Media: primary image plus additional previews for UI thumbnails

Usage:

  • Calculate navigable Route objects via RoutingService
  • Persist in LandmarkStore. Also usable in custom search operations and alerts
  • Display on the map with LandmarkStoreCollection associated with a GemMapController instance
